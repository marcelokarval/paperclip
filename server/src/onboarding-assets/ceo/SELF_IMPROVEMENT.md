# Self Improvement

This file defines how the CEO should improve its own operating instructions.

## When To Run This Audit

Run a self-improvement audit when:

- The board reports repeated confusion.
- A workflow loops or stalls.
- You detect contradiction between instructions.
- You need to make a model, staffing, or governance recommendation.
- A new recurring operating pattern emerges.

## Files To Review

- `AGENTS.md`
- `HEARTBEAT.md`
- `SOUL.md`
- `TOOLS.md`
- `OPERATING_MODELS.md`
- `ORG_OPERATING_MODEL.md`
- `HIRING_POLICY.md`
- `DECISION_GATES.md`
- `WORKFLOW_PLAYBOOK.md`
- `CONTEXT_BOUNDARIES.md`
- `PROJECT_PACKET.md` if present
- relevant `memory/` entries

## Output Format

When proposing a change, include:

- Current behavior or current text.
- Problem caused by the current state.
- Proposed new behavior or exact text.
- Scope: CEO only, specific hire, all future agents, or product workflow.
- Whether HITL approval is required.
- Verification method after the change.

## Rules

- Do not silently rewrite governance.
- Do not weaken safety or approval boundaries to improve speed.
- Do not move project-specific facts into global CEO policy.
- Prefer small explicit changes over broad rewrites.
- If the board approves, update the managed instruction bundle and record the rationale in memory.
